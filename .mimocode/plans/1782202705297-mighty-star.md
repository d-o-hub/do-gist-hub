# Add Dark Mode Toggle Button to Header

## Goal
Add a sun/moon icon button in the app header that toggles between light and dark mode.

## Files to Modify

### 1. `src/components/app.ts` — Add toggle button to header markup

**Header area (lines 393-397)**: Add a `<button id="theme-toggle">` before the sync indicator in `header-right`. Use inline SVG sun/moon icons (no external dependencies). Read current theme via `getTheme()` to set initial icon and `aria-label`.

- Import `getTheme` and `setTheme` from `../tokens/design-tokens`
- In `render()`: render button with sun icon (shows "switch to light") when dark, moon icon (shows "switch to dark") when light
- In event delegation: listen for `click` on `#theme-toggle` → call `setTheme(currentTheme === 'dark' ? 'light' : 'dark')`, re-render, dispatch `app:theme-change`

**No separate component needed** — keep it inline like other header elements.

### 2. `src/styles/navigation.css` — Style the toggle button

Add styles for `.theme-toggle` button matching existing `.icon-button` pattern. Use CSS `color-scheme` awareness so the button inherits theme colors.

### 3. `src/components/app.ts` — Wire up click handler

In the existing event delegation (click listener on `this.container`), add a case for `#theme-toggle`:
```ts
if (target.closest('#theme-toggle')) {
  const next = getTheme() === 'dark' ? 'light' : 'dark';
  setTheme(next);
  this.render(); // re-render to swap icon
}
```

## Icon Design
- **Dark mode active**: Sun icon (☀) — click to switch to light
- **Light mode active**: Moon icon (☽) — click to switch to dark
- Use inline SVG (24×24 viewBox) to avoid font/icon library dependency
- `aria-label` announces current state: "Switch to light mode" / "Switch to dark mode"

## Implementation Notes

- `setTheme('light'|'dark')` persists to `localStorage` and sets `data-theme` on `<html>` — existing infrastructure handles everything
- The button overrides any time/ambient/auto preference by setting an explicit `'light'` or `'dark'`
- Button is visible on all screen sizes (mobile + desktop)
- Follow existing `icon-button` class pattern for consistent styling
- No new files needed — all changes in existing files
