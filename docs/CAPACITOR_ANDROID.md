# Capacitor Android Setup & Play Store Deployment

## Prerequisites

- Node.js 24+, pnpm 9
- Java JDK 17+ (for local builds)
- Android Studio (for local development)
- `ANDROID_HOME` environment variable set
- GitHub account with repository access

---

## Initial Setup

Run these commands once to initialize the Android project:

```bash
# Build the web app first
pnpm run build

# Initialize Capacitor (if not already done)
npx cap init

# Add Android platform
npx cap add android

# Sync web assets to Android
pnpm run cap:sync
```

---

## Development Workflow

```bash
# Build and sync changes
pnpm run build:android

# Open Android Studio
pnpm run cap:android:open

# Run on connected device (adb)
npx cap run android
```

## GitHub Secrets Setup (Required for Release Builds)

To produce signed release builds in CI, you must configure **4 GitHub secrets** at **Settings → Secrets and variables → Actions**.

### Step 1: Generate a Keystore (in GitHub Codespaces)

The easiest way without installing Android Studio locally is to use GitHub Codespaces (JDK is pre-installed):

1. Go to your repository on GitHub → **Code** → **Codespaces** → **Create codespace on main**
2. In the terminal, generate a keystore:

```bash
keytool -genkey -v -keystore release-key.keystore -alias my-key-alias \
  -keyalg RSA -keysize 2048 -validity 10000 -storetype jks
```

3. You'll be prompted for:
   - **Keystore password** — create a strong password, save it somewhere safe
   - **Key password** — can be the same as the keystore password
   - Distinguished name fields (name, org, etc.) — press Enter to skip or fill in
   - Type `yes` to confirm

4. Convert the keystore to base64:

```bash
base64 -w 0 release-key.keystore > keystore_base64.txt
cat keystore_base64.txt
```

5. Copy the entire output string.

### Step 2: Add Secrets to GitHub

Create these 4 repository secrets:

| Secret Name | Value |
|---|---|
| `ANDROID_KEYSTORE_BASE64` | The full base64 string from `keystore_base64.txt` |
| `ANDROID_KEYSTORE_PASSWORD` | The keystore password you created |
| `ANDROID_KEY_ALIAS` | `my-key-alias` (or whatever alias you used) |
| `ANDROID_KEY_PASSWORD` | The key password (same as keystore password if you kept it) |

### Step 3: Verify

Push a `v*` tag to trigger the release workflow. The **"Decode Keystore and Build Release APK"** step in `.github/workflows/release.yml` will:

1. Decode the base64 keystore
2. Build a signed **release APK** (`app-release.apk`) for sideloading
3. Build a signed **Android App Bundle** (`app-release.aab`) for Play Store
4. Attach both to the GitHub Release

If the secrets are not configured, the step is skipped gracefully — debug APK is still published.

> **Security**: The keystore is decoded only during the CI run and discarded when the runner is torn down. Protect depends entirely on GitHub's secret storage. Rotate the keystore password annually.

## F-Droid Deployment (Free — Recommended)

> **No developer account fee.** F-Droid is a free, open-source app store.
> This is the recommended distribution channel for d.o. Gist Hub.

See [`docs/FDROID_DEPLOYMENT.md`](FDROID_DEPLOYMENT.md) for the complete guide.

### Quick Start

1. Push a `v*` tag to GitHub
2. Fork `https://gitlab.com/fdroid/fdroiddata`
3. Create `metadata/com.dogisthub.app.yml` (copy from `.fdroid.yml` in this repo)
4. Submit a Merge Request
5. App appears on F-Droid within 24-48 hours

### F-Droid vs Google Play

| Aspect | F-Droid (Recommended) | Google Play (Paid) |
|--------|-----------------------|--------------------|
| **Cost** | Free | $25 one-time fee |
| **Build** | From source (automated) | Upload AAB manually |
| **Open-source** | Required | Not required |
| **Proprietary deps** | Forbidden | Allowed |
| **Updates** | Auto via tags | Manual upload |

## Google Play Store Deployment (Requires $25 Account)

> **Note**: Requires a paid Google Play Console developer account ($25 one-time fee).
> Consider F-Droid (above) as the free alternative.

### Prerequisites

- [Google Play Console](https://play.google.com/console/) developer account ($25 one-time fee)
- A signed `app-release.aab` (generated by the release CI when secrets are configured)
- App screenshots (your Playwright screenshot pipeline in `playwright.config.ts` via `screenshots` project can generate these)
- Privacy policy URL (can use a GitHub Pages link)

### Step-by-Step

#### 1. Create Your App in Play Console

1. Go to [Google Play Console](https://play.google.com/console/)
2. Click **Create app**
3. Choose a name (e.g., "DO Gist Hub")
4. Select **App or game** → **App**
5. Choose **Free or Paid**
6. Accept the Declaration and click **Create app**

#### 2. Complete Store Listing

Fill in the Store listing section:

| Field | Notes |
|---|---|
| **Short description** | 80 characters max — "Offline-first GitHub Gist client" |
| **Full description** | 4000 characters max — features, offline capability, etc. |
| **Screenshots** | At least 2 phone + 2 tablet screenshots (use `pnpm run screenshots`) |
| **Feature graphic** | 1024×500 px banner image |
| **App icon** | 512×512 px icon (transparent PNG recommended) |
| **App category** | e.g., Productivity |
| **Tags** | e.g., GitHub, Gist, Offline |
| **Email** | Contact email for support |

> **Screenshot generation**: Use `playwright.config.ts` (screenshots project) to capture mobile/tablet screenshots via Playwright. Run `npx playwright test --project=screenshots` to generate them.

#### 3. App Signing by Google Play

During the first upload, Google will ask you to enroll in **Play App Signing**:
- Google manages the **app signing key** (used to sign APKs distributed by Play)
- Your keystore becomes the **upload key** (used to sign AABs you upload)
- Accept the default — Google generates the app signing key for you
- The **upload key certificate** (public) must be provided. Export it from your keystore:

```bash
keytool -export -rfc -alias my-key-alias -keystore release-key.keystore -file upload_cert.pem
cat upload_cert.pem
```

Copy the PEM output and paste it into the Play Console's "Upload key certificate" field.

#### 4. Upload Your AAB

1. Go to **Production** → **Create new release**
2. Upload your `android/app/build/outputs/bundle/release/app-release.aab` (download from GitHub Release artifacts)
3. Fill in release notes (auto-generated from your changelog)
4. Save and review

#### 5. Content Rating

1. Go to **Content Rating** → **Continue**
2. Fill out the questionnaire (for a developer tool/Gist client, this will be low maturity)
3. Save and submit

#### 6. Release Tracks (Recommended Progression)

| Track | Review Time | Testers Required | Purpose |
|---|---|---|---|
| **Internal Testing** | Minutes | Up to 100 emails | Quick smoke test with your team |
| **Closed Testing** | 1-3 days | Up to 100 users | QA with invited testers |
| **Open Testing** | 1-3 days | None (public) | Broader feedback before production |
| **Production** | A few hours to 2 days | None | Public release |

**Recommended flow:**
1. Start with **Internal Testing** — add your own email + developer emails
2. Promote to **Closed Testing** after basic verification
3. Once stable, promote to **Open Testing** or **Production**

#### 7. Pricing & Distribution

- Set **Free** for your app
- Select all countries, or a targeted subset
- Confirm that your app complies with Google's policies

### Post-Deployment

- **Update the App**: Push a new `v*` tag → CI builds new AAB → download from GitHub Releases → upload to Play Console → promote to production
- **Versioning**: `versionCode` auto-increments from `GITHUB_RUN_NUMBER`, `versionName` from the `VERSION` file
- **Monitoring**: Use the Play Console's **Android Vitals** dashboard for crash reports, ANRs, and performance

## Cloud Device Testing (Pre-Release)

Before shipping to Play Store, test on real devices in the cloud:

### Option 1: Firebase Test Lab (Recommended — Free Tier Available)

Firebase Test Lab runs your APK on real Google-hosted devices and automatically crawls the app looking for crashes.

**Setup:**

```bash
# Install Firebase CLI
npm install -g firebase-tools

# Login
firebase login

# Run a robo test (no test code needed — crawls automatically)
gcloud firebase test android run \
  --app android/app/build/outputs/apk/debug/app-debug.apk \
  --device model=Pixel7,version=34 \
  --device model=MediumTablet,version=34 \
  --timeout 5m
```

**Free tier**: 5 test runs/day. Enough for pre-release smoke tests.

### Option 2: BrowserStack App Live (Paid — ~$50/month)

- Real devices streamed via browser
- Manual interactive testing — swipe, tap, debug via Chrome DevTools remotely
- Best for manual QA before production release

### Option 3: Local USB Testing

```bash
# Connect Android device via USB with debugging enabled
npx cap run android
```

This is the fastest feedback loop and doesn't require any cloud services.

## Troubleshooting

### Cannot find Android SDK

```bash
export ANDROID_HOME=$HOME/Android/Sdk
export PATH=$ANDROID_HOME/platform-tools:$PATH
export PATH=$ANDROID_HOME/tools:$PATH
```

### Sync errors

```bash
# Clean and resync
rm -rf android
npx cap add android
pnpm run cap:sync
```

### Release build fails in CI (missing secrets)

The release APK/AAB step is **conditional** — if `ANDROID_KEYSTORE_BASE64` is empty, the step is skipped and only the debug APK is published. This is intentional for fork compatibility.

### AAB upload rejected by Play Console

- Ensure your keystore hasn't expired
- Verify the upload certificate matches what's registered in Play Console
- Check that `versionCode` is higher than the previous release (it will be, if `GITHUB_RUN_NUMBER` increments)
