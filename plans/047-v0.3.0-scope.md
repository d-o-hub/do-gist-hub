## Context

v0.2.0 stable shipped with:
- OAuth Device Flow (opt-in)
- Staleness indicators on gist cards
- Conflict-resolution E2E tests
- ProGuard CI smoke check
- Bundle budget CI enforcement
- AAB bundle build
- Firebase Test Lab CI step
- **F-Droid metadata** (`.fdroid.yml`) and deployment guide (`docs/FDROID_DEPLOYMENT.md`)
- Google Services dependency removed for F-Droid compliance

Plan 046 completed the release pipeline and Play Store documentation.
**Decision**: Use **F-Droid** (free) as the primary Android distribution channel
instead of Google Play Store (requires $25 paid account).

## Out of Scope (v0.4.0+)

| Item | Rationale |
|------|-----------|
| GitHub App installation tokens | Requires server with private key — contradicts ADR-005 (per ADR-028, deferred to v3) |
| Multi-account support | Architectural change — auth store assumes single user |
| Real-time collaboration | Backend sync server required |
| iOS packaging | No device access; Capacitor iOS setup deferred |
| Real Android hardware E2E tests | No physical device in current environment |
| Advanced conflict resolution (field-level merge) | Current local-wins/remote-wins/manual is sufficient |

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| F-Droid build server lacks Node.js/pnpm for Capacitor build | Add `prebuild` steps in `.fdroid.yml` to install Node.js and pnpm |
| F-Droid review cycle takes longer than expected | Start MR early; address Antifeature flags (NonFreeNet for GitHub API) |
| Google Play Store not available as fallback | GitHub Releases provide direct APK downloads for sideloading |
| OAuth telemetry reveals low adoption (users prefer PAT) | Keep both paths; PAT is cheaper to maintain |
| Mutation testing CI adds > 10 min to workflow | Run as optional check first, gate only after tuning |
| Bundle budget CI blocks legitimate PRs | Set reasonable thresholds; allow manual override with justification |
