## Context

Plans 039–044 completed all UI/UX modernization, SW TTL cache expiry, plan status hygiene, and the 038 audit sprints. The codebase is stable and healthy:

- 941+ tests passing, 91%+ line coverage
- Biome clean with strict rules (`noUnusedVariables`, `noExplicitAny`)
- All ADRs implemented or accepted
- SW cache with TTL expiry
- Cross-browser CI, Lighthouse CI, bundle budget checker all wired

However, two major gaps remain:

1. **Release v0.2.0** — VERSION is still `0.1.0` despite breaking changes (pnpm migration, Biome, TS 6, Vite 8, Capacitor 8). No release has been cut since v0.1.0.
2. **OAuth Device Flow** (ADR-028) — Adopted in principle but never implemented. The auth-proxy worker and client-side code don't exist. Users still must manually create and rotate PATs.

Additionally, several smaller audit items (F1, F3, F6 from plan 038) remain open.

---

## Goals

### Goal 1: Cut v0.2.0-rc.1 release

Bump VERSION, update CHANGELOG, tag and release.

### Goal 2: Implement OAuth Device Flow (ADR-028)

Create the auth-proxy Cloudflare Worker (`auth-proxy/worker.ts`) and client-side OAuth flow code to allow users to sign in with GitHub credentials instead of manual PAT creation.

### Goal 3: Close remaining audit gaps

- **F1**: Conflict-resolution UI E2E tests
- **F3**: Staleness indicators on gist cards
- **F6**: ProGuard CI delta verification smoke check
- Wire existing `check-bundle-size.sh` into CI

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| OAuth Device Flow is discouraged by GitHub for phishing | In-app warnings: "Only enter code on github.com"; app name + icon shown in consent screen |
| Worker proxy becomes single point of failure | PAT path remains as fallback; users can self-host the 80-line worker |
| v0.2.0-rc.1 may surface regressions | Run full Playwright suite (3 shards) + cross-browser smoke before tag |
| Staleness indicator may expose stale data UX | Use `lastSyncedAt` from IndexedDB metadata; graceful fallback if unavailable |
